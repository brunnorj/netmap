Configurações de Sincronização na Nuvem (Supabase)

Conecte sua aplicação a um banco de dados relacional Supabase de forma segura e descentralizada.

🔑 Credenciais de Conexão

Estes dados são salvos localmente de forma privada no seu navegador. O código-fonte da aplicação permanece limpo e seguro.

Status da Conexão:

🔄 Central de Sincronização de Dados

Exportar para Nuvem (Backup)

Envia todas as tabelas locais (computadores, switches, tomadas, VLANs, logs e usuários) para o seu banco Supabase, substituindo os dados existentes lá.

Importar da Nuvem (Restaurar)

Recupera todos os dados salvos na nuvem do Supabase e substitui o banco de dados do seu navegador atual.

Estrutura SQL Necessária (Copie e Cole no SQL Editor do Supabase)

-- execute este script no sql editor do supabase para criar as tabelas
create table if not exists netmap_vlans (
  id integer primary key,
  name text,
  subnet text,
  gateway text,
  "totalIps" integer
);

create table if not exists netmap_switches (
  id bigint primary key,
  name text unique,
  model text,
  "totalPorts" integer,
  location text
);

create table if not exists netmap_outlets (
  id double precision primary key,
  name text unique,
  location text,
  "switchName" text,
  port integer,
  status text
);

create table if not exists netmap_devices (
  id bigint primary key,
  name text,
  ip text unique,
  "vlanId" integer,
  "switchName" text,
  port integer,
  outlet text
);

create table if not exists netmap_users (
  id bigint primary key,
  username text unique,
  password text,
  role text
);

create table if not exists netmap_logs (
  id double precision primary key,
  timestamp text,
  "user" text,
  role text,
  action text,
  details text
);
